# Receive Coaching Local Coach

A fully local, multi-coach CLI app powered by a local LLM via Ollama.

- **43 coaches** across practical, psychological, wellbeing, and spiritual domains
- **Per-coach RAG:** drop `.txt`/`.md` files (and `.pdf`, if `pypdf` is installed) in any coach's `sources/` folder and the coach grounds its questions in that material via embeddings
- **Per-user memory** tagged by coach — shared profile, separate sessions per coach
- **Pattern detection** on user input (15 configured patterns incl. avoidance, overthinking, self-doubt), with a crisis handoff: high-risk messages route to human-support helplines, even when the model is down
- **Dependency-free CLI** — `receive_coach.py` is Python stdlib only (the optional desktop UI backend uses `fastapi` + `uvicorn`)
- **Fallback** to a rule-based coach if the model server is unreachable (marked `[fallback: rule-based]`; the reason is shown in Settings and logged)

## Quick start

**Portable .exe (no install):** double-click `dist/ReceiveCoaching-portable-1.0.0.exe`. Single-file Electron app. Extracts to a temp folder on each launch, writes user data to `%APPDATA%\receive-coaching-ui\`. Build yourself with `npm run build` (see "Packaging" section).

**Desktop app from source:** double-click `coach-ui.bat`. Opens the same native window, runs from source without packaging. First run installs Electron (~1 min).

**CLI:** double-click `coach.bat`, or from any terminal in this folder type `coach` (after running `install-alias.ps1` once).

**Otherwise, the underlying commands are:**

```powershell
python .\receive_coach.py                                  # default or last used coach
python .\receive_coach.py --coach business                 # specific coach
python .\receive_coach.py --list-coaches                   # print all 43 coaches
python .\receive_coach.py --coach-info keto                # print a coach's full config
```

The app connects to Ollama at `http://127.0.0.1:11434/v1` by default. It expects `llama3.1` pulled (`ollama pull llama3.1`) and `nomic-embed-text` for retrieval (`ollama pull nomic-embed-text`).

If Ollama isn't running, the app silently falls back to a rule-based coach and prints `[fallback: rule-based ...]` next to the coach name so you know.

## Desktop UI (recommended)

A native Electron app with:

- **Left sidebar** — all 43 coaches, searchable, with live source-count indicators
- **Chat window** — message bubbles, streaming tokens as they arrive, thinking indicator during model warmup, **Stop** button mid-stream, **Regen** on the latest reply (regenerating doesn't double-record the session in memory), **Edit** on your latest message, **Say** to read any reply aloud with your local Windows voices
- **Chat history** — "New chat" archives the thread (per coach, last 20); the History panel restores or deletes archived chats
- **Right drawer** — Memory panel (sessions, patterns, commitments, profile, a **mood-over-time strip**, and an on-demand **Insights report** generated by your local model) and Sources panel (drag-drop files incl. PDF, delete, re-index; 5 MB per file)
- **Settings modal** — backend URL, embed config, user switching, a **model picker** (override every coach's model from a live list of what your backend has pulled), auto-read toggle, and the last model error (so a config problem doesn't masquerade as "Ollama not running")
- **Voice input (optional)** — run a local whisper.cpp server and set `RECEIVE_COACH_STT_URL`; a mic button appears in the composer. Audio never leaves your machine.
- **Auto-spawns the Python backend** on launch, on a random free port, secured with a per-session auth token; kills it cleanly on close and shows a banner if it dies

**Launch:** double-click `coach-ui.bat`. First run installs Electron (~1 min; uses existing Node.js). Subsequent runs open in seconds.

**Requirements (running from source):**
- Python 3 with `fastapi` and `uvicorn` installed (`pip install fastapi uvicorn`; add `pypdf` for PDF sources)
- Node.js (already on your system)
- Ollama running with `llama3.1` and `nomic-embed-text` pulled, OR raw llama.cpp running (see below)

**Requirements (packaged .exe):** just Ollama/llama.cpp — the backend interpreter is bundled.

**Power users:** `npm start` for the same, or `npm run dev` with `--dev` flag for DevTools.

### Packaging as a portable .exe

Single-file portable `.exe` (no install, no registry entries). Uses `electron-builder`.

```powershell
npm run build          # produces dist/ReceiveCoaching-portable-1.0.0.exe
```

The built .exe:
- Bundles Electron + the UI + a **self-contained backend** (`server.exe`, built from `server.py` + `receive_coach.py` via PyInstaller — no Python needed on the host) + configs + all 43 coaches
- Only external requirement: Ollama (or llama.cpp) running with the models pulled
- `npm run build` chains both stages automatically (`prebuild` → PyInstaller → electron-builder); building from a fresh clone needs `pip install pyinstaller fastapi uvicorn pypdf` once
- Falls back to system Python (`python server.py`) if the bundled backend is missing, so from-source runs are unaffected
- On first launch, seeds a writable copy of `coaches/` into `%APPDATA%\receive-coaching-ui\coaches\`
- Memory, logs, and embedding caches go to `%APPDATA%\receive-coaching-ui\`
- `build/icon.ico` is the app icon; regenerate with `python build/make_icon.py` if you change the design

**One-time setup if building on Windows without admin / Developer Mode:**

`electron-builder`'s `winCodeSign` extraction fails on benign macOS symlinks (the archive includes darwin `.dylib` files that need admin to symlink). To work around this, run once after `npm install`:

```powershell
python build/setup_7za_wrapper.py
```

This builds a small PyInstaller wrapper around 7-Zip that swallows those specific errors, installs it into `node_modules/7zip-bin/win/x64/7za.exe`, and backs up the original as `7za-real.exe`. Requires `pyinstaller` (`pip install pyinstaller`). Only needed once per `node_modules/` (re-run after wiping).

## Adding a new coach

```
coaches/<name>/
  coach.json          # name, display_name, description, model, system_prompt
  sources/            # optional .txt / .md files to ground the coach
  .embeddings.json    # auto-generated cache
```

Template `coach.json`:
```json
{
  "name": "your_coach",
  "display_name": "Your Coach",
  "description": "One-line description shown in /coach list.",
  "model": "llama3.1",
  "system_prompt": "You are a ..."
}
```

Then use with `--coach your_coach` or `/coach your_coach` mid-session.

## Grounding a coach in your own material (RAG)

1. Drop `.txt` or `.md` files into `coaches/<name>/sources/`. PDFs work too if `pypdf` is installed (`pip install pypdf`) — text is extracted at index time; without pypdf, `.pdf` files are simply ignored and the CLI stays stdlib-only.
2. Run the app (or `/reindex` mid-session) — it chunks the files, embeds each chunk via Ollama's `nomic-embed-text`, and caches embeddings keyed by file path + content hash.
3. On every user turn, the coach pulls the top 3 most relevant chunks (cosine ≥ 0.35) and grounds its questions in that material.

For docx and other formats, extract the text first (copy-paste is fine).

## CLI flags

- `--user <id>` — user id for memory storage (default: `justin`)
- `--coach <name>` — coach to use. If omitted, uses the user's last coach or `general`.
- `--list-coaches` — print all coaches and exit
- `--coach-info <name>` — print a coach's full config (incl. system prompt) and exit

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `RECEIVE_COACH_BASE_URL` | `http://127.0.0.1:11434/v1` | Chat server URL (OpenAI-compat `/v1/chat/completions`) |
| `RECEIVE_COACH_EMBED_MODEL` | `nomic-embed-text` | Embedding model name (used as cache key) |
| `RECEIVE_COACH_EMBED_FORMAT` | *(auto)* | `ollama` or `openai`. Defaults to auto-detect |
| `RECEIVE_COACH_API_KEY` | *(unset)* | Only needed if fronting a server with auth |
| `RECEIVE_COACH_AUTH_TOKEN` | *(unset)* | Bearer token the UI backend (`server.py`) requires on every request when set. The Electron launcher generates and wires this automatically; set it manually only if you run `server.py` yourself and want it protected. Unset = auth off (dev). |
| `RECEIVE_COACH_STT_URL` | *(unset)* | Optional local speech-to-text endpoint for voice input, e.g. `http://127.0.0.1:8091/inference` from `whisper-server -m ggml-base.en.bin --port 8091`. Must be a `127.0.0.1` URL (the UI's CSP blocks anything else). Unset = mic button hidden. |

Per-coach chat model is set inside that coach's `coach.json` (`model` field).

## Using raw llama.cpp instead of Ollama

Both backends are supported. The embeddings code auto-detects which one you're running against. Installed via `winget install ggml.llamacpp`.

**1.** Edit `start-llama-server.bat` to point `MODEL=` at your GGUF file, then double-click (or run it).

**2.** In a second terminal:
```powershell
$env:RECEIVE_COACH_BASE_URL = "http://127.0.0.1:8090/v1"
$env:RECEIVE_COACH_EMBED_FORMAT = "openai"
coach
```

**Pitfalls you'll hit** (so you don't):

- `--pooling mean` is **required** when using a chat model (Llama, Mistral, Qwen, etc.) for embeddings. The default pooling is not OpenAI-compatible and returns a 400. Already included in `start-llama-server.bat`.
- Embeddings from a chat model in mean-pool mode are *functional but not ideal* — a dedicated embedding model (nomic-embed, bge-small, etc.) is better. For best retrieval, run a second `llama-server` on a different port with a dedicated embedding GGUF, and keep the chat server on port 8090. Most people won't bother; the mean-pool chat model works.
- The embeddings cache (`coaches/*/.embeddings.json`) is keyed by embedding-model name and vector dim. Switching backends auto-invalidates and re-embeds — you don't have to delete anything manually.

**Why you might prefer Ollama:** zero config, bundled `nomic-embed-text`, model management via `ollama pull`, lighter memory footprint in most cases.

**Why you might prefer raw llama.cpp:** direct GGUF file control, specific quantizations, lower-level flags, no Go server in front.

## Privacy

The `.gitignore` excludes:
- `data/users/*.json` — your personal memory (never commit)
- `logs/*` — conversation logs (never commit)
- `coaches/*/.embeddings.json` — large machine-specific caches
- `.env` / `.env.*` / `*.key` / `*.pem` — env files and keys

Do not share these files. They contain your private conversations.

Memory files are written atomically (temp file + rename), so a crash mid-save can't
truncate them. If a memory file is ever unreadable, it's moved aside as
`data/users/<user>.corrupt-<timestamp>.json` and a fresh one is started — nothing is
deleted silently.

## Safety notes

**Crisis handoff:** messages matching the `crisis_or_high_risk` pattern (self-harm signals, configured in `config/pattern_detection_rules.json`) route to human-support resources — Samaritans 116 123 (UK), 988 (US/CA), Lifeline 13 11 14 (AU), findahelpline.com — instead of normal coaching. This works even in rule-based fallback mode when no model is running.

Several coaches operate in clinical-adjacent domains. Each has explicit scope limits in its system prompt:

- **cbt, rebt** — not a substitute for therapy; refer out for clinical conditions
- **addiction_recovery** — support only, for people already in recovery; does not replace 12-step, therapy, medical treatment
- **anxiety_support** — not for clinical anxiety disorders; refer to CBT/exposure therapy for those
- **keto** — general education only; contraindications listed in the coach; not for diabetes on meds, kidney disease, eating disorder history, etc.
- **health_nutrition, wellness, sleep** — general guidance; refer out for medical conditions
- **grief, divorce, parenting, caregiving** — supportive; refer to therapists for abuse, trauma, complicated grief
- **law_of_attraction, universal_laws, abundance_manifestation** — explicitly refuse "you manifested your own suffering" framings
- **shamanic_life** — frames itself as shamanic-influenced coaching, not indigenous shamanism; acknowledges roots with respect; does not use plant medicines

Each coach can refer to peer coaches when a user's need is outside its scope.
